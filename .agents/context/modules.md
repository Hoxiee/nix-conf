# Modules

## hardware.nix

Generated by `nixos-generate-config`. Lives only on the target machine.
Never create it. Never remove its import from `hosts/<n>/default.nix`.

```nix
imports = [
  ./hardware.nix  # on machine, not in repo — correct by design
];
```

## One concern per module

Each file has exactly one responsibility.
If new functionality does not fit an existing module, create a new file.
Do not extend an unrelated module to avoid creating a new file.

## No hostname conditionals inside modules

Wrong:
```nix
services.pipewire.lowLatency.enable =
  if config.networking.hostName == "laptop" then true else false;
```

Right — put it in `hosts/laptop/default.nix`:
```nix
services.pipewire.lowLatency.enable = true;
```

## GPU modules

`gpu/amd.nix` → imported only in `hosts/laptop/default.nix`.
`gpu/nvidia.nix` → imported only in `hosts/desktop/default.nix`.
Neither references the other. Neither checks hostname.

All NVIDIA Wayland environment variables live in `gpu/nvidia.nix` only.
`niri.nix` must remain GPU-agnostic.

## niri.nix — portal stack

Must configure the full portal stack. Do not defer this.

Required:
- `xdg-desktop-portal` with `xdg-desktop-portal-gtk` as fallback
- `xdg-desktop-portal-wlr` for screen sharing
- `dbus` session bus enabled
- Environment variables via `environment.sessionVariables`
  or `systemd.user.sessionVariables`

Screen sharing, file picker, and auth dialogs must work out of the box.

## sddm.nix

SilentSDDM is a flake input. Do not vendor its files into this repo.
Theme wiring lives in `modules/nixos/sddm.nix`.

## docker.nix

Opt-in via option. Default: disabled. Default mode: rootless.

```nix
options.modules.docker.enable = lib.mkEnableOption "Docker";
```

## flatpak.nix

Enables Flatpak system-wide and adds Flathub remote.
Individual apps are not managed declaratively — users install manually.

## base.nix (nixos)

Owns: locale, time, fonts, users, base packages.
Nerd Fonts declared here under `fonts.packages`. Do not scatter font declarations.

Base packages (do not remove without reason):
`git`, `neovim`, `ripgrep`, `fd`, `bat`, `eza`, `fzf`, `htop`,
`wget`, `curl`, `unzip`, `zip`, `file`, `which`, `tree`, `jq`

## zsh.nix (home)

Owns: Zsh via `programs.zsh`, Starship via `programs.starship`,
common aliases and environment variables.
Starship config sourced from `config/starship.toml` — do not inline as attrset.

## Dotfile linking pattern

```nix
# modules/home/alacritty.nix
xdg.configFile."alacritty/alacritty.toml".source = ../../config/alacritty.toml;
```

Use `xdg.configFile` consistently. Do not mix with `home.file` without reason.

## Options vs plain assignments

Use `lib.mkOption` only when the module exposes tuneable behaviour.
Plain attribute assignment is correct when there is nothing to tune.
Do not wrap simple config in options to look architectural.

## Display-specific values

| Host | Resolution | Notes |
|------|------------|-------|
| laptop | 2560×1600 | Fractional scaling needed |
| desktop | 3440×1440 @ 165 Hz | Single ultrawide |

Niri output config, waybar dimensions, fuzzel sizing that differ per host →
`users/<username>/laptop.nix` or `users/<username>/desktop.nix`.
Never in shared modules.
